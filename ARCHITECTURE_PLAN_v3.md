## 🧠 ЭТАП 1 — Консилиум (Multi-Agent System)

**Статус: ✅ РЕАЛИЗОВАН**

### Агенты (3x DeepSeek V3.2)
```
IMPULSE  → Агрессивный скальпер (M5/M15, паттерны свечей, Stochastic)
TREND    → Консервативный тренд  (H1/H4, MA, ADX, Ichimoku)
ANALYST  → Price Action аналитик (Уровни, RSI-дивергенция, Brooks)
```

### Раунды принятия решений
```
R1: Независимые голоса (агенты не видят друг друга)
R2: Дебаты (видят позиции коллег, могут изменить мнение)
R3: Вето (новости, спред, корреляционный барьер)
R4: Взвешенный вердикт (большинство ≥ 67%, уверенность ≥ MIN_CONFIDENCE)
```

### Фильтры перед входом
- Высокорисковые новости (30 мин до)
- Спред > 3× MAX_SPREAD_PIPS
- Корреляционное противоречие (разница > 6 голосов)
- Временной штраф (ИИ знает убыточные часы суток)

## 🔍 ЭТАП 2 — ИИ-Аудитор (Post-Trade Analysis)

**Статус: ✅ РЕАЛИЗОВАН (auditor_agent.py)**

### Метрики по каждой сделке
| Метрика | Описание |
|---|---|
| **MFE** | Maximum Favorable Excursion — сколько пунктов цена прошла в нашу сторону |
| **MAE** | Maximum Adverse Excursion — насколько сделка уходила в минус |
| **TTE** | Time To Exit — сколько свечей/минут до закрытия |
| **Efficiency** | Какой % от MFE мы захватили через TP |
| **Noise Ratio** | MAE / SL_dist — был ли стоп выбит шумом |

### Классификация сделок
| Класс | Условие | Действие |
|---|---|---|
| `PERFECT_WIN` | MAE<15% SL, Eff≥80% | Запомнить паттерн в alpha_patterns.json |
| `WASTED_WIN` | MFE > TP×2.5 | Предложить трейлинг / больший TP |
| `EFFICIENT_WIN` | Нормальная победа | Записать, анализировать |
| `NOISE_STOP` | noise_ratio>0.7, затем цена пошла в нашу сторону | Предложить шире SL |
| `WRONG_DIRECTION` | MAE > SL×1.5 сразу | Пересмотреть логику входа |
| `NORMAL_LOSS` | Обычный стоп | Просто записать |

### Синтез паттернов
- Минимум **3 повторения** одинакового паттерна для подтверждения
- Паттерн идентифицируется по: **сессия + направление + RSI-зона**
- Подтверждённые паттерны читаются **перед каждым решением** Консилиума

## ⚗️ КАСТОМНЫЕ ИНДИКАТОРЫ (Фирменные, нигде не описаны)

**Статус: ✅ РЕАЛИЗОВАН (custom_indicators.py)**

### 1. PGI — Pressure Gradient Index
```
Идея: традиционные индикаторы измеряют движение цены.
      PGI измеряет АСИММЕТРИЮ ДАВЛЕНИЯ внутри свечи
      сразу на 4 таймфреймах.

Формула:
  pressure = (close - low) / (high - low)   ← "где закрылась цена внутри свечи"
  PGI = EMA( pressure × volume_ratio, 14 )
  PGI_gradient = линейный наклон PGI за 5 точек

Сигнал:
  PGI > 0.65 + gradient > 0.3  → накопление, вероятен рост     (+3 BUY)
  PGI < 0.35 + gradient < -0.3 → распределение, вероятно падение (+3 SELL)

Уникальность:
  Конвергенция PGI на M15 + H1 + H4 одновременно = 
  редкий и точный сигнал, невидимый стандартным RSI/MACD
```

### 2. TVA — Temporal Volatility Asymmetry
```
Идея: кто агрессивнее — быки или медведи?
      Не направление движения, а РАЗМАХ движения по типу свечи.

Формула:
  ATR_bull = средний True Range только БЫЧЬИХ свечей (close > open)
  ATR_bear = средний True Range только МЕДВЕЖЬИХ свечей
  TVA = ATR_bull / ATR_bear

Дополнение:
  TVA по часам суток (hourly_profile) — в 9:00 быки агрессивнее,
  в 15:00 медведи. Это знание подтверждает или опровергает сигнал.

Сигнал:
  TVA > 1.25 → быки доминируют  (+2 BUY)
  TVA < 0.80 → медведи доминируют (+2 SELL)
```

### 3. CandleDNA — Свечной Геном
```
Идея: каждая свеча — это 6-ген "ДНК":
  G1: размер тела / ATR (0-5)
  G2: направление (-1, 0, +1)
  G3: верхняя тень / диапазон (0-3)
  G4: нижняя тень / диапазон (0-3)
  G5: позиция закрытия в high-low (0-4)
  G6: volume ratio (0-4)

Последовательность 3 свечей = 18-ген "цепочка"
Хэш цепочки → поиск в памяти → winrate по исходам

Если паттерн встречался ≥3 раз с WR≥60% → сигнал активен

Уникальность:
  Обучается на реальных данных пользователя.
  Распознаёт последовательности, которые никакой учебник не описывал.
  Работает как "персональная ДНК-библиотека" стратегии.
```

**Статус: ✅ РЕАЛИЗОВАН**

### Файлы знаний
```
knowledge/
  strategy_handbook.txt    ← Статические правила (Nison, Brooks)
  alpha_patterns.json      ← Динамические паттерны от аудитора
  candle_dna_memory.json   ← База CandleDNA (до 2000 паттернов)
```

### Жизненный цикл паттерна
```
1. Аудитор замечает PERFECT_WIN/EFFICIENT_WIN
2. Извлекает паттерн из ответа ИИ
3. Сохраняет в alpha_patterns.json (unconfirmed)
4. При 3+ повторениях → помечает confirmed = true
5. Консилиум читает alpha_patterns ПЕРЕД каждым решением
6. Паттерны без повторений через 100 новых записей — удаляются
```

```

```
